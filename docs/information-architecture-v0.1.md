# GatherUp v0.1 页面信息架构

## 1. 总体结构

GatherUp v0.1 分成四个主要区域：

- 活动广场：用户浏览公开活动。
- 场地库：用户查询各城市可办活动的场地经验。
- 我的活动：用户管理自己的报名、订单和历史活动。
- 工作台：用户创建和管理自己的活动。
- 活动管理台：活动创建者处理单个活动的报名、付款、选座、通知和导出。

第一版要避免页面过多，但每个核心流程必须完整。重点是让组织者少切工具，让参与者少问问题。

账号采用统一模型：同一个用户既可以参与活动，也可以创建活动。“参与者”和“组织者”不是两种账号，而是用户在具体活动里的关系。

## 2. 顶层导航

### 未登录

- 登录/注册

v0.1 原型阶段，核心内容都需要登录后查看。未登录用户先进入登录/注册页。

### 已登录用户

- 活动广场
- 场地库
- 我的活动
- 工作台
- 个人中心

用户可以查看历史参与活动、待付款活动、待选座活动和已结束活动，也可以从工作台创建新活动。

## 3. 活动广场

### 目标

让参与者快速找到可以报名的公开活动。

### 页面内容

- 顶部筛选区：
  - 城市
  - 活动场景
  - 活动状态
  - 时间范围
- 活动列表：
  - 活动名称
  - 活动场景
  - 城市
  - 时间
  - 场地
  - 单人价格
  - 剩余名额
  - 状态标签
- 空状态：
  - 当前筛选条件下暂无活动
  - 提示清空筛选

### 活动卡片字段

- 封面或色块占位。
- 活动名称。
- 时间和城市。
- 场地。
- 价格。
- 剩余名额。
- 状态标签。

### v0.1 暂不做

- 个性化推荐。
- 关注组织者。
- 活动评论。
- 复杂排序。

## 3.1 场地库

### 目标

让组织者快速查找哪些影院、咖啡馆和会议空间可以承接活动，以及过往组织者的体验如何。

### 页面内容

- 顶部筛选区：
  - 城市
  - 场地类型
  - 可办状态
  - 关键词
- 场地列表：
  - 场地名称
  - 城市和区域
  - 场地类型
  - 可办状态
  - 适合活动类型
  - 容量备注
  - 评分和经验数量
  - 最近验证日期

### 场地详情页

- 场地名称。
- 城市、区域和地址。
- 可办状态。
- 适合活动类型。
- 容量。
- 价格备注。
- 联系建议。
- 组织者经验。
- 优势和注意事项。
- 体验评分。

### v0.1 暂不做

- 用户提交新场地。
- 场地经验审核。
- 场地方认领。
- 场地官方报价管理。

## 4. 活动详情页

### 目标

让参与者判断是否报名，并清楚知道下一步。

### 页面内容

- 活动名称。
- 活动状态。
- 时间、城市、场地、地址。
- 单人费用。
- 剩余名额。
- 是否允许多人报名。
- 报名截止时间。
- 活动说明。
- 组织者说明。
- 最新通知预览。

### 主要操作

- 登录并报名。
- 查看我的报名。

### 状态变化

- 未登录：显示登录后报名。
- 已登录且未报名：显示报名按钮。
- 已报名未付款：显示去付款。
- 已付款待审核：显示等待审核。
- 已确认付款未选座：显示去选座。
- 已选座：显示我的座位。

## 5. 报名流程

### 第一步：填写报名信息

字段：

- 昵称。
- 联系方式类型。
- 联系方式。
- 人数。
- 同行人 GatherUp ID。
- 是否接受候补。
- 给组织者的备注。

规则：

- 不允许多人报名时，人数固定为 1。
- 允许多人报名时，人数不能超过组织者设置的每单上限。
- 同行人 ID 数量应等于人数减 1。
- 同行人 ID 是否必须已注册，作为待确认问题保留。

### 第二步：生成订单

提交成功后生成：

- 订单编号。
- 应付金额。
- 报名状态。
- 付款状态。

### 第三步：进入付款截图页

显示：

- 订单编号。
- 应付金额。
- 付款说明。
- 上传付款截图入口。

## 6. 付款流程

### 参与者侧

状态：

- 待上传截图。
- 待组织者审核。
- 已确认。
- 已驳回。

参与者操作：

- 上传付款截图。
- 重新上传付款截图。
- 查看审核结果。

### 组织者侧

付款管理表字段：

- 订单编号。
- 昵称。
- 人数。
- 应付金额。
- 付款截图。
- 付款状态。
- 提交时间。
- 确认时间。
- 备注。

组织者操作：

- 查看大图。
- 确认付款。
- 驳回付款。
- 标记退款。
- 添加备注。

## 7. 选座流程

### 前置条件

只有付款状态为 `confirmed` 的报名可以选座。

### 参与者侧

内容：

- 座位图。
- 座位状态说明。
- 当前已选座位。
- 确认选座按钮。

规则：

- 需要选择的座位数等于报名人数。
- 已分配座位不可选。
- 锁定座位不可选。
- 同一订单下的多个座位可以不连续。

### 组织者侧

内容：

- 座位图。
- 已选座位详情。
- 未选座订单列表。
- 座位状态筛选。

操作：

- 锁定座位。
- 解锁座位。
- 手动调整座位。
- 清空某个订单的座位。

## 8. 我的活动

### 目标

让参与者看到自己历史参与过的活动。

### 页面分组

- 待付款。
- 待审核。
- 待选座。
- 即将开始。
- 已结束。
- 已取消。

### 活动记录字段

- 活动名称。
- 订单编号。
- 时间。
- 城市。
- 报名状态。
- 付款状态。
- 座位状态。
- 快捷操作。

## 9. 个人中心

### 页面内容

- 头像。
- 昵称。
- GatherUp 公开 ID。
- ID 剩余修改次数。
- 绑定方式：
  - 微信
  - 邮箱
  - 手机号
- 历史参与活动入口。
- 组织工作台入口。

### 公开 ID 规则

- 每个用户有一个唯一公开 ID。
- 公开 ID 用于多人报名时填写同行人。
- 公开 ID 最多修改两次。
- 修改次数用完后不可再修改。

## 10. 组织者工作台

### 目标

让组织者看到自己创建的活动和需要处理的任务。

### 页面内容

- 创建活动按钮。
- 活动列表。
- 待处理事项：
  - 待审核付款。
  - 未选座订单。
  - 候补报名。
  - 临近开始活动。

### 活动列表字段

- 活动名称。
- 活动状态。
- 报名人数。
- 已付款人数。
- 待审核付款数。
- 已选座人数。
- 开始时间。
- 快捷操作。

## 11. 创建活动

### 表单分组

基础信息：

- 活动名称。
- 活动场景。
- 流程模板。
- 细分标签。
- 城市。
- 场地/影院名称。
- 地址。
- 日期和时间。

报名规则：

- 人数上限。
- 报名截止时间。
- 是否允许多人报名。
- 每单最多人数。
- 是否接受候补。

付款规则：

- 单人费用。
- 付款说明。

订单规则：

- 订单编号格式。
- 订单编号前缀或活动代码。

可见性：

- 进入活动广场。
- 仅链接可见。

座位设置：

- 排数。
- 每排座位数。
- 不可用座位。

说明：

- 活动说明。
- 组织者联系说明。

## 12. 活动管理台

### 概览

- 总报名数。
- 已付款人数。
- 待审核付款。
- 已选座人数。
- 剩余名额。
- 活动公开链接。
- 下一步建议。

### 报名名单

- 查看报名。
- 修改报名状态。
- 查看同行人 ID。
- 添加备注。

### 付款管理

- 审核截图。
- 确认付款。
- 驳回付款。
- 标记退款。

### 座位管理

- 查看座位图。
- 锁定座位。
- 调整座位。
- 查看未选座名单。

### 通知

- 发布通知。
- 保存草稿。
- 复制通知文案。

### 导出

- 导出报名 CSV。
- 导出付款 CSV。
- 导出座位 CSV。
