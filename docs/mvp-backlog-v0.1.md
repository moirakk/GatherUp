# GatherUp v0.1 MVP 开发任务清单

## 1. 目标

这份清单用于把 GatherUp v0.1 拆成可执行开发任务。

优先级原则：

- 先跑通核心闭环。
- 先跑通基础活动组织闭环，线下观影作为第一个强样例，但不把平台限制在线下观影。
- 先做 Web App，移动端适配。
- 先使用付款截图确认，不接官方支付。

## 2. Milestone 0：项目初始化

目标：搭好真实应用骨架。

任务：

- 初始化 Next.js + TypeScript 项目。
- 配置 Tailwind CSS。
- 建立基础布局。
- 建立莫兰迪色系设计 token。
- 建立基础组件目录。
- 配置 ESLint/Prettier。
- 配置 Supabase 客户端。
- 添加环境变量示例。

完成标准：

- 本地可以启动开发服务器。
- 首页可以显示活动广场空壳。
- 基础样式和组件可复用。

## 3. Milestone 1：账号和用户基础

目标：用户可以登录，并拥有 GatherUp 公开 ID。

任务：

- 实现登录页。
- 先接入邮箱登录。
- 预留微信和手机号入口。
- 创建用户资料表。
- 生成默认公开 ID。
- 实现公开 ID 修改。
- 限制公开 ID 最多修改两次。
- 实现个人中心。

完成标准：

- 用户可以登录。
- 用户可以看到自己的公开 ID。
- 用户最多只能修改两次公开 ID。

## 4. Milestone 2：活动创建和活动广场

目标：组织者可以创建活动，公开活动能进入活动广场。

任务：

- 创建 `events` 表。
- 实现创建活动页。
- 支持活动基础信息。
- 支持活动场景。
- 支持流程模板。
- 支持报名规则。
- 支持付款规则。
- 支持订单编号格式选择。
- 支持座位排数和每排座位数。
- 实现活动广场。
- 实现活动详情页。
- 支持公开/仅链接可见。

完成标准：

- 组织者可以创建一场活动，并选择活动场景和流程模板。
- 公开活动能在活动广场出现。
- 仅链接活动不出现在活动广场。

## 5. Milestone 3：报名和订单

目标：参与者可以报名，并生成订单。

任务：

- 创建 `registrations` 表。
- 创建 `registration_attendees` 表。
- 创建 `event_order_counters` 表。
- 实现报名页。
- 根据活动规则控制是否显示多人报名。
- 多人报名时记录同行人 GatherUp ID。
- 生成订单编号。
- 计算应付金额。
- 创建付款记录。
- 实现我的活动列表。

完成标准：

- 参与者可以报名活动。
- 报名后生成订单编号。
- 多人报名规则生效。
- 我的活动中能看到订单。

## 6. Milestone 4：付款截图和审核

目标：参与者上传付款截图，组织者确认付款。

任务：

- 创建 `payments` 表。
- 创建 `payment_proofs` 表。
- 配置 Supabase Storage。
- 实现付款截图页。
- 实现图片上传。
- 实现付款状态：未上传、待审核、已确认、已驳回、已退款。
- 实现组织者付款管理页。
- 支持查看截图大图。
- 支持确认付款。
- 支持驳回付款。

完成标准：

- 参与者可以上传付款截图。
- 组织者可以确认或驳回付款。
- 付款确认前不能选座。

## 7. Milestone 5：座位

目标：付款确认后参与者可以选座。

任务：

- 创建 `seats` 表。
- 创建 `seat_assignments` 表。
- 创建活动时生成座位。
- 实现参与者选座页。
- 实现组织者座位管理页。
- 支持锁定/解锁座位。
- 支持按订单分配座位。
- 防止重复占座。
- 报名取消后释放座位。

完成标准：

- 付款确认后才能选座。
- 选择座位数必须等于报名人数。
- 同一个座位不能被多个订单选择。

## 8. Milestone 6：组织者管理台

目标：组织者可以完整管理活动。

任务：

- 实现组织工作台。
- 实现活动管理台概览。
- 实现报名名单页。
- 实现付款管理页。
- 实现座位管理页。
- 实现基础通知页。
- 实现 CSV 导出页。

完成标准：

- 组织者可以看到待审核付款、未选座订单、报名人数。
- 组织者可以处理报名、付款、座位。
- 组织者可以导出名单。

## 9. Milestone 7：运营完善

目标：让 v0.1 更像可试用产品。

任务：

- 空状态。
- 加载状态。
- 错误提示。
- 移动端适配。
- 表格响应式处理。
- 关键操作二次确认。
- 基础可访问性检查。
- README 更新。

完成标准：

- 参与者在手机上能完成报名、付款、选座。
- 组织者在桌面端能完成核心管理流程。
- 页面没有明显文字溢出或重叠。

## 10. 暂不进入 v0.1

- 官方支付。
- 微信小程序。
- 更多活动场景和模板组合。
- 分时段预约。
- 套餐库存。
- 活动推荐算法。
- 活动评论。
- 关注组织者。
- 多管理员细粒度权限。
