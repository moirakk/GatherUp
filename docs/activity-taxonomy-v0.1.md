# GatherUp v0.1 活动类型体系

## 1. 设计原则

GatherUp 不应该把平台边界锁死在线下观影或生咖。

更合适的设计是把活动拆成两层：

- 活动场景：用户理解“这是什么活动”。
- 流程模板：系统理解“这个活动需要哪些组织能力”。

这样 GatherUp 可以覆盖大到校园活动、官方会议，小到好友私下聚会，同时仍然保持第一版开发可控。

## 2. 活动场景

活动场景用于活动广场筛选、活动详情展示、用户心智分类。

v0.1 建议先提供这些场景：

- 同好活动：线下观影、生咖、粉丝聚会、读书会、桌游局。
- 校园活动：社团活动、讲座、迎新、比赛、校园市集。
- 会议会务：发布会、沙龙、论坛、内部会议、行业交流。
- 好友聚会：聚餐、旅行、生日局、私下小聚。
- 工作坊：课程、训练营、手作、体验课。
- 快闪/市集：快闪店、市集摊位、品牌活动。

后续可以继续加：

- 公益活动。
- 展览活动。
- 体育活动。
- 亲子活动。
- 企业团建。

## 3. 流程模板

流程模板用于决定活动创建时默认启用哪些模块。

v0.1 建议先抽象为：

- 基础报名：报名表、人数上限、名单管理。
- 报名收款：报名、付款截图、付款确认。
- 选座活动：报名、付款确认、座位图、选座。
- 签到活动：报名、名单、签到码/核销。
- 分时预约：时间段、名额、预约名单。
- 记录型聚会：参与记录、费用记录、照片/备注，适合好友私下聚会。

第一版真实开发可以先实现：

- 基础报名。
- 报名收款。
- 选座活动。

分时预约、签到活动和记录型聚会可以作为 v0.2 之后继续扩展。

## 4. 场景和模板的关系

场景不等于模板。

例子：

- 线下观影：同好活动 + 选座活动。
- 生咖：同好活动 + 分时预约/报名收款。
- 校园讲座：校园活动 + 基础报名/签到活动。
- 官方会议：会议会务 + 报名收款/签到活动。
- 好友聚餐：好友聚会 + 记录型聚会。
- 手作课：工作坊 + 报名收款/签到活动。

这样同一个场景可以使用不同模板，同一个模板也可以服务多个场景。

## 5. 数据字段建议

`events` 表不应该只保留一个 `type` 字段。

建议改成：

- `category`：活动场景。
- `template`：流程模板。
- `custom_type_label`：组织者自定义显示名。

示例：

- `category = community`
- `template = seating`
- `custom_type_label = 线下观影`

活动广场优先展示 `custom_type_label`，没有时展示 `category` 的中文名。

## 6. v0.1 前端筛选

活动广场第一批筛选建议：

- 全部活动。
- 同好活动。
- 校园活动。
- 会议会务。
- 好友聚会。

不要把“线下观影/生咖”作为顶层筛选。它们可以作为活动卡片里的细分标签出现。
