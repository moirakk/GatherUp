# GatherUp

GatherUp 是一个面向线下活动的综合组织平台。

它希望覆盖从校园活动、官方会议到好友私下聚会、同好活动的全流程组织，帮助用户在同一个平台里完成数调、地点投票、报名、付款截图确认、选座、通知、签到和导出名单。

当前项目处于 v0.1 原型阶段：先用 Next.js + mock 数据跑通核心流程和页面体验，后续再接入真实账号、数据库、文件上传和支付相关能力。

## 当前状态

- 已搭建 Next.js / App Router / TypeScript Web App。
- 已实现统一账号登录原型：同一个账号既可以参与活动，也可以创建活动。
- 已实现活动广场、活动详情、我的活动、订单详情、工作台、创建活动、活动管理台。
- 已实现活动 ID 和多组织者绑定原型。
- 已实现组织者财务中心原型：费用模式、收入、支出、结余和凭证状态。
- 已实现组织者宣传发布中心：活动链接、数调入口、报名入口、宣传文案和物料清单。
- 已实现组织者通知中心原型：报名通知、付款提醒、选座通知、成团通知和活动当日通知。
- 已实现场地情报库原型：城市、场地类型、可办状态和组织者经验。
- 已实现数调、地点投票、报名、付款截图占位、付款后选座、订单下一步动作等核心流程的前端原型。
- 暂未接入真实数据库、真实验证码、图片上传、正式支付和正式权限系统。

## 本地运行

安装依赖后运行：

```bash
npm install
npm run dev:webpack -- --hostname 127.0.0.1 --port 3000
```

然后打开：

```text
http://127.0.0.1:3000
```

说明：当前项目在本地预览时建议优先使用 `dev:webpack`，比默认 Turbopack dev 模式更稳定。

## 演示账号

当前登录只是原型模拟，账号信息保存在浏览器 cookie 中。

```text
邮箱：miki@gatherup.local
密码：gatherup123
```

登录后可以进入活动广场、我的活动和工作台。GatherUp 不把账号拆成“参与者账号”和“组织者账号”，同一个用户可以在不同活动里扮演不同角色。

## 重要产品决策

- 第一版先做中文界面，后续再扩展英文、日文、韩文等语言。
- 第一版先做 Web App，并适配移动端。
- 所有核心内容都需要登录后查看。
- 账号是统一账号，不区分参与者账号和组织者账号。
- 全球账号底座优先使用邮箱，Google / Apple 作为快捷登录，微信作为中国区增强入口。
- 活动创建者拥有自己活动的管理权限。
- 第一版付款方式为“付款截图 + 组织者确认”，暂不接官方支付。
- 必须先付款并由组织者确认，之后才能选座。
- 活动支持免费、收费、AA 记账三种费用模式。
- 收入、支出、预计结余需要在组织者财务中心里清楚展示。
- 每个活动需要公开活动 ID，方便搜索、分享和现场核对。
- 同一个活动可以绑定多个组织者。
- 场地库是平台级模块，用于沉淀各城市可办活动的影院、咖啡馆和其他空间。
- 是否允许多人报名由活动创建者决定。
- 多人报名时需要记录同行人的 GatherUp ID。
- GatherUp ID 最多允许修改两次。

## 静态原型

```bash

```
